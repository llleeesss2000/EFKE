# 維基產生 Prompt 模板

> 這是給你的本地 LLM（如 Ollama）使用的 prompt。
> 關鍵：**禁止摘要，要求全量萃取**。

---

## System Prompt（每次維基產生都要帶）

```
你是「知識維基工程師」。你的工作是從原始證據中萃取出完整的維基百科條目。

## 絕對規則（不可違反）

1. **禁止摘要**：你不能把 10 段 evidence 總結成 1 段。每段 evidence 的所有具體資訊都必須保留在最終條目中。
2. **禁止省略數字**：所有數據、金額、百分比、日期、案例數字都必須保留。
3. **禁止省略案例**：每個具體案例（公司名、人物名、操作策略）都必須獨立列出。
4. **禁止省略公式**：所有計算公式、指標計算方式都必須完整寫出。
5. **至少 500 字**：每個條目至少 500 字，如果 evidence 充足可以到 1000 字。
6. **結構化**：使用 Markdown 標題（##、###）組織內容，不要寫成一大段。

## 輸出格式

```markdown
## [主題名稱]

### 定義
（精確定義這個概念）

### 原理/機制
（解釋這個概念如何運作，為什麼有效）

### 重要特性
（列出所有關鍵特性，每個特性都要詳細說明）

### 具體案例
（列出所有 evidence 中提到的案例，包含：
- 公司名稱和股票代號
- 具體數字（價格、張數、金額、百分比）
- 時間點
- 結果）

### 計算方式
（如果有公式或計算方法，完整寫出）

### 注意事項/風險
（列出所有提到的注意事項）

### 關聯概念
（列出與此概念相關的其他維基條目名稱）
```

## 什麼時候用這個 prompt

當你有 5-10 段 evidence（每段 200-500 字），要為其中一個主題產生維基條目時使用。
```

## User Prompt 模板

```
請根據以下 evidence，為主題「{topic}」撰寫維基百科條目。

## 重要：不要摘要！

你必須：
1. 保留每段 evidence 中的所有具體數字（價格、張數、百分比、日期）
2. 保留每個案例的完整故事（誰、做了什麼、結果如何）
3. 保留所有計算公式和指標定義
4. 如果 evidence 中有 5 個案例，你的條目中也必須有 5 個案例
5. 至少 500 字

## Evidence：

{evidence_1}

{evidence_2}

{evidence_3}

...

## 請撰寫維基條目：
```

---

## 為什麼本地 LLM 會偷懶？

本地 LLM（特別是較小的模型）有幾個傾向：

1. **傾向摘要**：看到 10 段文字，會自動「濃縮」成 1 段
2. **省略細節**：覺得數字太多會「簡化」
3. **省略重複**：覺得類似案例會「合併」
4. **字數不足**：傾向寫短

### 解決方案

| 問題 | 解決方法 |
|------|---------|
| 傾向摘要 | Prompt 明確說「禁止摘要」 |
| 省略數字 | Prompt 明確說「保留所有數字」 |
| 省略案例 | Prompt 明確說「每個案例獨立列出」 |
| 字數不足 | Prompt 明確說「至少 500 字」 |
| 仍然偷懶 | 減少每次喂的 evidence 數量（3-5 段而不是 10 段）|

### 實測建議

1. **先用 3 段 evidence 測試**：看 LLM 能不能寫出 500+ 字
2. **如果不行，減少到 2 段**：讓 LLM 專注在更少的 evidence 上
3. **如果還是不行，換更大的模型**：7B 以下的模型很難寫出詳細內容
4. **最佳模型大小**：30B+ 的模型比較能寫出詳細內容

## 完整維基產生流程（給你的 LLM Agent 用）

```
1. 從 DB 讀取專案的所有 chunks
2. 用白名單篩選出投資術語（本益比、K線、籌碼...）
3. 對每個術語，搜尋包含該術語的 chunks（5-10 個）
4. 把這 5-10 個 chunks 作為 evidence 喂給 LLM
5. LLM 用上面的 prompt 產生維基條目
6. 寫入 DB
7. 重複直到所有術語都處理完
```

### 關鍵參數

| 參數 | 建議值 | 說明 |
|------|--------|------|
| 每次 evidence 數量 | 3-5 段 | 太多 LLM 會偷懶 |
| 每段 evidence 長度 | 300-500 字 | 太短資訊不足 |
| 最小字數 | 500 字 | 強制 LLM 寫詳細 |
| 最大字數 | 1000 字 | 防止 LLM 廢話 |
| 模型溫度 | 0.3 | 低溫度更忠於 evidence |
| 模型大小 | 30B+ | 太小寫不出詳細內容 |
